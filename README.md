# File transfer
اسم المشروع
File-transfer — أداة بسيطة لنقل الملفات بين المتصفحات والأجهزة مع دعم صفحات عميل P2P وتوليد رمز QR واختبارات السرعة.

نبذة سريعة
حلاً خفيفاً يعتمد على Node.js لتقديم واجهة خفيفة في المتصفح (client-p2p.html) لنقل الملفات، مع أدوات مساعدة لتوليد رموز QR ثابتة (generate-static-qr.js)، واختبارات/تحسينات للسرعة (speed-test.js, speed-optimizations.js). المجلد downloads مخصّص لتخزين التنزيلات/الملفات المارة عبر الخادم أو الواجهة.

الميزات الرئيسية
- واجهة عميل P2P للمتصفح (client-p2p.html) لنقل الملفات بين الأجهزة.
- خادم Node.js بسيط لتقديم الملفات والصفحات (server.js).
- أداة لتوليد رموز QR ثابتة لتسهيل مشاركة روابط الجلسة أو ملفات (generate-static-qr.js).
- ملفات CSS للاستجابة (responsive.css) لواجهة الويب.
- أدوات لاختبار وتحسين الأداء (speed-test.js و speed-optimizations.js).
- مجلد downloads لحفظ الملفات المنقولة/التحميلات.

المتطلبات
- Node.js (نسخة حديثة — v14+ أو أعلى موصى بها)
- npm

الاستعداد من المستودع (سريع)
1. استنساخ المستودع:
   git clone https://github.com/sdevp-tech/File-transfer.git
   cd File-transfer

2. تثبيت الاعتماديات:
   npm install

تشغيل الخادم (الأقصر)
- من مسار المشروع بعد التثبيت:
  node server.js
- افتح المتصفح وانتقل إلى:
  http://localhost:3000
  (إذا كان الخادم مُكوّناً على منفذ مختلف فاستخدم المنفذ المناسب — غالباً يمكن تغيير ذلك بواسطة متغير بيئة PORT أو إعداد داخل server.js)

ملاحظات عامة عن التشغيل
- إذا احتوى package.json سكربت start، يمكنك تشغيل:
  npm start
- صفحة العميل P2P موجودة كـ client-p2p.html — يمكنك فتحها مباشرة في المتصفح أو الوصول إليها عبر الخادم لتحصل على نفس الوظائف مع إمكانية تبادل بيانات الجلسة عبر الخادم.

توليد رمز QR ثابت
- يوجد ملف generate-static-qr.js ضمن المشروع. يُستخدم لتوليد رمز QR لروابط أو مسارات معينة — يمكن تشغيله عبر Node.js:
  node generate-static-qr.js <URL>
- قد ينتج صورة ملف أو يطبع بيانات الـ QR في مخرجاته حسب إعداد السكربت. راجع السكربت لتفاصيل الخيارات (حجم، نوع المخرجات، مسار الحفظ).

اختبارات السرعة وتحسين الأداء
- المشروع يحتوي على speed-test.js و speed-optimizations.js لاختبار وتحسين أداء نقل الملفات (قياس throughput، قطع/تجزئة، إعدادات TCP/WebRTC إن وُجدت).
- لتشغيل اختبارات بسيطة:
  node speed-test.js
- اقرأ محتوى الملفات لفهم المعلمات القابلة للتعديل (حجم العيّنات، عدد الاتصالات، الخ).

هيكل الملفات (مُجمّع من محتوى المستودع)
```text
README.md                  ← هذا الملف (اقتراح لنسخة مُحدّثة)
client-p2p.html            ← واجهة عميل P2P في المتصفح
generate-static-qr.js      ← أداة توليد رمز QR ثابت
responsive.css             ← أنماط CSS للاستجابة
server.js                  ← خادم Node.js الرئيسي (خدمة الصفحات/واجهات API)
speed-optimizations.js     ← نصائح/أدوات لتحسين السرعة
speed-test.js              ← سكربت لاختبار سرعة النقل
downloads/                 ← مجلد لحفظ الملفات المنقولة/التحميلات
package.json               ← تعريف الحزم والسكربتات (تثبيت/تشغيل)
package-lock.json          ← قفل الإصدارات
node_modules/              ← حزم مسنودة (لا تُضمّن عادة في المستودعات الناتجة)
```

كيف تعمل المكونات معاً (ملخّص تنفيذي)
- server.js يعمل كخادم HTTP بسيط يقدم صفحات الويب (مثل client-p2p.html) و/أو نقاط نهاية API مساعدة.
- client-p2p.html هي واجهة تعمل في المتصفح وتوفّر تجربة إرسال/استقبال ملفات — عادة تستخدم قنوات مباشرة (مثل WebRTC) أو تبادل بيانات عبر الخادم لبدء جلسة نقطة لنقطة.
- للاتصال السريع بين جهازين يمكن إنشاء رابط جلسة أو صفحة تحتوي على رمز QR تم توليده بواسطة generate-static-qr.js لكي يمسحها الجهاز الآخر وينضم للجلسة.
- مجلد downloads يُستخدم لتخزين الملفات المستقبلة أو المؤقتة.

التكوين (اقتراحات)
- متغيرات بيئة مقترحة:
  - PORT — منفذ تشغيل الخادم (مثال: 3000)
  - HOST — عنوان الاستماع (مثال: 0.0.0.0)
  - HTTPS_* أو مفاتيح SSL — إذا رغبت بتشغيل الخادم عبر HTTPS لتمكين WebRTC/ميزات آمنة
- تعامل مع إعدادات CORS إذا كنت تستخدم الخادم كبنية وسيطة لبدء اتصالات P2P عبر نطاقات مختلفة.

مقترحات لتحسين وتجهيز المنتج للإنتاج

- تمكين HTTPS وتهيئة شهادة صالحة (Let's Encrypt).

- اختبار تحميل مع سيناريوهات متعددة العملاء للتحقق من قيود الباندويث.

استكشاف المشكلات الشائعة
- الخادم لا يبدأ: تأكد من تثبيت الاعتماديات (npm install) وتشغيل node server.js أو npm start. افحص سجل الأخطاء في الطرفية.
- صفحة العميل لا تتصل بنجاح: تحقق من أن عنوان الخادم ومنفذه صحيحان، وأن المتصفح يسمح بالاتصالات المطلوبة (خصوصاً عند استخدام WebRTC قد تحتاج HTTPS).
- رمز QR لا يعمل: تأكد من أن الرابط الذي ولّدته قابل للوصول من الجهاز الذي يمسح الرمز (مثال: عنوان محلي غير متاح عبر شبكة خارجية).

أسئلة مقترحة يمكنك طرحها بعد قراءة الكود
- هل server.js يحتوي على نقطة إشارات (signaling) WebSocket أو يستخدم خدمة خارجية لبدء جلسات P2P؟
- كيف تُدار الملفات في مجلد downloads — هل تُحذف تلقائياً أم تبقى إلى أن يحذفها المستخدم؟
- ما إعدادات توليد QR (حجم الصورة، نوع الملف، مسار الحفظ) في generate-static-qr.js؟
